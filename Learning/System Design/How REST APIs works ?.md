REST (Representational State Transfer) is an architectural style for designing networked applications. It uses HTTP protocols to perform CRUD operations on resources.


| Constraint    | Description                                                          |
| ------------- | -------------------------------------------------------------------- |
| Client-Server | Separation of concerns - UI and data storage are independent         |
| Stateless     | Each request contains all info needed; server stores no client state |
| Cacheable     | Responses must define themselves as cacheable or not                 |
